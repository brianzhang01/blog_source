Model Selection for Binary Image Data

January 1, 1

Introduction

This exercise is about performing Bayesian model selection over different Bernoulli generative models. We use a dataset of binary images of digits, as part of remotely working through the 2017 Probabilistic and Unsupervised Learning course at Gatsby.

Read our data

First let’s read in our data (available for download from the course website).

digits = as.matrix(read.table('binarydigits.txt'))
dim(digits)
## [1] 100  64

The original assignment contains some visualization code in Matlab and Python, which we port to R (code hidden).

plot_all(digits)

Let’s also generate some other fantasy data which will be useful for comparison. The datasets will be of the same size, except one will contain random noise drawn from \(\text{Bern}(0.5)\), and the other from \(\text{Bern}(0.1)\).

set.seed(2)
container = list()
container[[1]] = digits
container[[2]] = matrix(as.numeric(runif(6400) < 0.5), nrow=100, ncol=64)
plot_all(container[[2]])

container[[3]] = matrix(as.numeric(runif(6400) < 0.1), nrow=100, ncol=64)
plot_all(container[[3]])

Derive expressions

The marginal likelihood is obtained by marginalizing the likelihood over the parameter space. It allows for comparison between different models in the Bayesian setting, and serves as an alternative to cross-validation.

Let our data matrix be denoted \(Y_{ij}\), with \(1 \leq i \leq n = 100\), the number of images, and \(1 \leq j \leq m = 64\), the number of pixels per image. We neglect any spatial structure in the image, instead treating each image as a vector of length \(m = 64\). All the entries are binary, \(Y_{ij} \in \{0, 1\}\).

The original assignment asks us to compare between the following three models to see which best explains the data:

  1. \(M_1\): each pixel is generated independently from a fair coin flip, \(Y_{ij} \sim \text{Bern}(0.5)\) i.i.d.
  2. \(M_2\): each pixel is generated independently from a Bernoulli distribution with same but unknown parameter \(\theta\). We place a uniform prior on \(\theta\) and write \(p(\theta) = \text{Unif}(0, 1)\), \(p(Y_{ij} | \theta) = \text{Bern}(\theta)\) i.i.d.
  3. \(M_3\): each of the \(m = 64\) pixel positions are described as following their own Bernoulli process, each with an unknown independent \(\theta_j\). Now we have \(p(\boldsymbol{\theta}) = p(\theta_1) \cdots p(\theta_m)\), \(p(\theta_j) = \text{Unif}(0, 1)\), and \(p(Y_{ij} | \boldsymbol{\theta}) = \text{Bern}(\theta_j)\), with independent observations.

In the most general case, Bayesian model selection places priors over the models. However, in our case, we opt for a uniform prior, \(p(M_1) = p(M_2) = p(M_3) = 1/3\). Thus we only compare between the marginal likelihoods.

Model 1

Under \(M_1\), there are no parameters to marginalize over. Instead: \[ p(Y | M_1) = \prod_{i,j}p(Y_{ij} | M_1) = \prod_{i,j}0.5=(0.5)^{6400} \]

Model 2

Let \(N_1\) be the number of 1’s and \(N_0\) be the number of 0’s among the collective data entries \(Y_{ij}\). Note that \(N_0 + N_1 = nm = 6400\).

For an individual Bernoulli data point \(y \sim \text{Bern}(\theta)\), the likelihood is \(\theta\) if \(y=1\) and \(1-\theta\) if \(y=0\). This is often combined to write the likelihood as \(p(y|\theta) = \theta^y (1-\theta)^{1-y}\). For our dataset of \(nm\) independent Bernoulli data points, the joint likelihood is: \[ \begin{align*} p(Y|\theta, M_2) &= \prod_{i,j} p(Y_{ij}|\theta, M_2) = \prod_{i,j} \theta^{Y_{ij}} (1-\theta)^{(1-Y_{ij})} = \theta^{(\sum_{ij}Y_{ij})}(1-\theta)^{(N-\sum_{ij}Y_{ij})}\\ &= \theta^{N_1} (1-\theta)^{N_0} \end{align*} \] To find the marginal likelihood, we integrate out \(\theta\) from the joint likelihood. First we have: \[ p(Y|M_2) = \int p(Y|\theta, M_2)p(\theta| M_2) d\theta = \int p(Y|\theta, M_2) \cdot 1 \cdot d\theta \] since our prior over \(\theta\) is uniform. Plugging in our earlier expression: \[ \begin{align*} p(Y|M_2) &= \int p(Y|\theta, M_2) d\theta = \int_0^1 \theta^{N_1} (1-\theta)^{N_0} d\theta\\ &= B(N_1+1, N_0+1) \int_0^1\frac{1}{B(N_1+1, N_0+1)}\theta^{N_1} (1-\theta)^{N_0} d\theta\\ &= B(N_1+1, N_0+1)\\ &= \frac{N_1!N_0!}{(N_1+N_0+1)!} \end{align*} \] Here, \(B(a, b)\) is the beta function, and we have used the fact that the \(\text{Beta}(N_1+1, N_0+1)\) distribution integrates to 1.

Model 3

Under \(M_3\), our joint likelihood is: \[ \begin{align*} p(Y|\boldsymbol{\theta}, M_3) &= \prod_{i,j} p(Y_{ij} | \boldsymbol{\theta}, M_3) = \prod_{i,j} p(Y_{ij} | \theta_j) = \prod_{i,j} \theta_j^{Y_{ij}} (1-\theta_j)^{(1-Y_{ij})}\\ &= \prod_{j} \theta_j^{N_{1j}} (1-\theta_j)^{{N_{0j}}} \end{align*} \] Here we define \(N_{1j}\) and \(N_{0j}\) as the number of entries among \(Y_{ij}\) that are 1 and 0, respectively. For instance, with \(i=3\), this counts the number of 1’s and 0’s among pixel position 3, across the \(n=100\) images.

The marginal likelihood is \[ \begin{align*} p(Y|M_3) &= \int p(Y|\boldsymbol{\theta}, M_3) p(\boldsymbol{\theta} | M_3) d\boldsymbol{\theta} = \int p(Y|\boldsymbol{\theta}, M_3) \cdot 1 \cdot d\boldsymbol{\theta} \\ &= \int \left(\prod_{j} \theta_j^{N_{1j}} (1-\theta_j)^{{N_{0j}}}\right) d\boldsymbol{\theta}\\ &= \prod_{j} \left( \int_0^1 \theta_j^{N_{1j}} (1-\theta_j)^{{N_{0j}}} d\theta_j \right)\\ &= \prod_{j} \frac{N_{1j}!N_{0j}!}{(N_{1j}+N_{0j}+1)!} \end{align*} \] where we have used the beta integral trick from the previous section.

Resulting code

For computation, we use the negative log of the marginal likelihood, both to avoid numerical overflow / underflow and for easier display. The corresponding expressions are easily derived: \[ \begin{align*} -\ln p(Y|M_1) &= 6400 \cdot \ln2\\ -\ln p(Y|M_2) &= -[\ln(N_1!) + \ln(N_0!) - \ln((N_1+N_0+1)!)]\\ -\ln p(Y|M_3) &= -\sum_j[\ln(N_{1j}!) + \ln(N_{1j}!) - \ln((N_{1j}+N_{0j}+1)!)] \end{align*} \]

We approximate the log factorial using the highly accurate Ramanujan formula.

logfact = function(n) {
  if (n < 10) { return(log(factorial(n))) }
  return(n*log(n) - n + log(n*(1 + 4*n*(1 + 2*n)))/6 + 0.5*log(pi))
}

nlls = function(digits) {
  n1 = sum(digits)
  n0 = length(digits) - sum(digits)
  
  # case of p = 0.5
  nll1 = -((n0 + n1) * log(0.5))
  
  # case of fixed, unknown p
  nll2 = -(logfact(n1) + logfact(n0) - logfact(n1+n0+1))
  
  # case of per-pixel p
  nll3 = 0
  for (j in 1:64) {
    n1j = sum(digits[,j])
    n0j = length(digits[,j]) - n1j
    nll3 = nll3 - (logfact(n1j) + logfact(n0j) - logfact(n1j+n0j+1))
  }
  return(c(nll1, nll2, nll3))
}

Run the code and produce graphs

Now we run our code and produce a ggplot bar graph collecting all the results (plotting code hidden). We show the negative log-likelihood per bit of data (dividing by 6400), so that lower bars are better.

nlls(container[[1]])
## [1] 4436.142 4283.721 3851.196
nlls(container[[2]])
## [1] 4436.142 4439.387 4545.206
nlls(container[[3]])
## [1] 4436.142 2087.394 2212.776

Some conclusions:

  • For the original data, with structured patterns among the pixels, the per-pixel Bernoulli model performs best.
  • For synthetic data with \(p = 0.5\) for all data points, the corresponding \(p = 0.5\) Bernoulli model performs best, with the model of uniform but unknown \(p\) close behind.
  • Finally, for synthetic data with \(p = 0.1\) for all data points, the model of uniform but unknown \(p\) performs best.
  • For all data, the model of \(p = 0.5\) gives same likelihood, which is equal to the marginal likelihood.

Note that if we instead compared the three models based on the maximum likelihood, we would always get \(M_3 \geq M_2 \geq M_1\) (greater meaning better performance), because the models have increasing expressivity. In contrast, the above results show that the marginal likelihood metric is able to effectively penalize overly complex models.

Further reading

This exercise illustrates model selection under a simple case of comparing between Bernoulli models. There is some helpful exposition out there on motivating the marginal likelihood for Bayesian model selection, terming it a “Bayesian Occam’s razor.” For more information, I recommend:

  • Section 5.3 of Kevin Murphy’s Machine Learning: A Probabilistic Perspective, first edition (2012), on “Bayesian model selection.”
  • Section 3.4 of Christopher Bishop’s Pattern Recognition and Machine Learning on “Bayesian Model Comparison.”
  • Chapter 28 of David J.C. MacKay’s Information Theory, Inference, and Learning Algorithms, titled “Model Comparison and Occam’s Razor.”
  • Wikipedia page on the Bayes factor.
